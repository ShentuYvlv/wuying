# 联调踩坑

## 1. 地域填错

问题：

- `DescribeAndroidInstances` 返回空
- 明明实例存在，但代码提示 `Instance not found`

解决：

- `WUYING_REGION_ID` 必须和控制台实例地域一致
- 当前实例实际在 `cn-hangzhou`

## 2. eds-aic endpoint 不能按杭州直连

问题：

- 直接访问 `eds-aic.cn-hangzhou.aliyuncs.com` 出现 TLS / 解析异常

解决：

- 中国区业务地域仍走上海控制面 endpoint
- 业务地域用 `BizRegionId`
- 当前代码已默认处理这个映射

## 3. 共享网络不能本机直连

问题：

- 共享网络实例只给私网 ADB
- 本机执行 `adb connect 10.x.x.x:5555` 失败

解决：

- 共享网络只能在能访问对应 VPC 私网的机器上连
- 如果要本机直连，需要改成 VPC 网络实例组

## 4. VPC 网络一键 ADB 不是创建完就能连

问题：

- 控制台一键 ADB 已创建
- `adb connect 公网IP:端口` 仍失败

解决：

- 必须放行安全组入方向 `TCP 5555/5555`
- 之后再执行 `adb connect <公网IP>:<公网端口>`

## 5. adbkey 放错位置

问题：

- 控制台已绑定密钥
- ADB 连接仍鉴权失败或无效

解决：

- `adbkey` 至少要能被本地 `adb.exe` 读取
- Windows 官方默认路径是 `%USERPROFILE%\.android\adbkey`
- 当前项目也支持显式配置 `ADB_VENDOR_KEYS`

## 6. platform-tools 不完整

问题：

- `adb.exe` 存在，但 `adb version` 没输出或直接退出

解决：

- 需要完整的 Android `platform-tools`
- 不能只保留 `adb.exe`

## 7. AccessKey 被拒绝

问题：

- OpenAPI 返回 `NotAuthorizedApi`
- 或 `InvalidAccessKeyId.AccessPolicyDenied`

解决：

- 手动 ADB 模式下直接跳过 OpenAPI
- 依赖：
  - 控制台手动创建 ADB
  - `.env` 填 `WUYING_MANUAL_ADB_ENDPOINT`

## 8. 已经连上 ADB，但脚本还是停在找输入框

问题：

- `adb connect` 成功
- `uiautomator2` 报输入框选择器超时

解决：

- 先确认豆包是否真的启动到前台
- 用以下命令检查当前前台页面：

```powershell
.\platform-tools\adb.exe -s 106.14.114.146:100 shell dumpsys window | findstr "mCurrentFocus mFocusedApp"
```

- 如果前台还是 `com.android.launcher3`，说明：
  - 豆包没装
  - 或包名写错

## 9. 默认豆包包名不一定对

问题：

- 脚本默认写的是 `com.larus.nova`
- 实际设备上可能不是这个包名

解决：

- 先查真实安装包：

```powershell
.\platform-tools\adb.exe -s 106.14.114.146:100 shell pm list packages | findstr /i "doubao nova larus"
```

- 再把 `.env` 里的 `DOUBAO_PACKAGE_NAME` 改成真实值

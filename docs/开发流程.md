# 开发流程

## 运行入口

统一入口只有一个：

```powershell
.\venv\Scripts\python.exe .\run_app.py --platform doubao --prompt "你好"
```

参数：

- `--platform`
  - 平台名
- `--prompt`
  - 提问内容
- `--instance-id`
  - 可选，覆盖 `.env` 里的默认实例

## 新增平台流程

以后新增 `deepseek` / `kimi` / `qianwen` / `yuanbao`，按下面做。

### 1. 配置

文件：

- [config.py](E:/all code/C一念/wuying/src/wuying/config.py)

要做：

- 加平台包名
- 加启动页
- 加选择器配置
- 加输出目录配置

### 2. 新建工作流

目录：

- `src/wuying/application/workflows/`

做法：

- 新建一个平台工作流文件
- 继承 `ChatAppWorkflow`
- 只写平台差异逻辑

必须实现：

- `__init__`
- `_collect_extra_metadata`

必要时覆写：

- `_ensure_new_chat_session`
- `_ensure_chat_input_ready`
- `_is_chat_page`

### 3. 注册平台

文件：

- [platform_registry.py](E:/all code/C一念/wuying/src/wuying/application/platform_registry.py)

要做：

- 注册平台名
- 注册工作流类
- 写平台描述

### 4. 联调

先做这几步：

1. 手工 `adb connect`
2. 手工确认包名
3. 手工确认能启动 App
4. 导出页面层级
5. 填选择器
6. 跑 `run_app.py`

### 5. 验证通过标准

至少验证：

1. 不在目标 App 前台时，能自动拉起
2. 已在旧聊天页时，能新建会话
3. 能切到文本输入
4. 能发送问题
5. 能拿到回答
6. 结果 JSON 正常落盘

## 改代码时的原则

- 入口只改接口层
- 业务编排只改应用层
- 外部工具调用只改调用层
- 不要把平台差异逻辑写进入口
- 不要把平台差异逻辑写进调用层
- 公共逻辑优先放 `base.py`
- 平台特殊逻辑只放各自 workflow
